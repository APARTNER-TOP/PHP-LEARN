Як використовувати AJAX в PHP та jQuery

Сьогодні ми збираємося дослідити концепцію AJAX з PHP. Техніка AJAX допомагає вам покращити користувальницький інтерфейс вашої програми та покращити загальне сприйняття кінцевого користувача.

Що таке AJAX?

AJAX розшифровується як асинхронний JavaScript та XML і дозволяє асинхронно отримувати контент із внутрішнього сервера без оновлення сторінки. Це дозволяє оновлювати вміст веб-сторінки без перезавантаження.

Давайте розглянемо приклад, щоб зрозуміти, як ви можете використовувати AJAX у повсякденній розробці програм. Скажімо, ви хочете створити сторінку, яка відображає інформацію профілю користувача, з різними розділами, такими як особиста інформація, соціальна інформація, повідомлення, повідомлення та інше.

Звичайний підхід полягає у створенні різних веб-сторінок для кожного розділу. Так, наприклад, користувачі клацають посилання соціальної інформації, щоб перезавантажити браузер і відобразити сторінку із соціальною інформацією. Це уповільнює навігацію між розділами, оскільки користувачеві доводиться чекати перезавантаження браузера та повторного відображення сторінки щоразу.

З іншого боку, ви можете використовувати AJAX для створення інтерфейсу, який завантажує всю інформацію без оновлення сторінки. У цьому випадку ви можете відобразити різні вкладки для всіх розділів, і, натиснувши на вкладку, він витягує відповідний вміст із внутрішнього сервера та оновлює сторінку, не оновлюючи браузер. Це допоможе покращити загальне сприйняття кінцевого користувача.

Загальний виклик AJAX працює приблизно так:

Давайте швидко пройдемося по звичайному потоку AJAX: Спочатку користувач зазвичай відкриває веб-сторінку синхронним запитом.
Потім користувач натискає елемент DOM - зазвичай кнопку чи посилання - який ініціює асинхронний запит до серверу. Кінцевий користувач не помітить цього, оскільки виклик виконується асинхронно та не оновлює браузер. Однак, ви можете розпізнати ці AJAX-дзвінки за допомогою такого інструменту, як Firebug.
У відповідь на запит AJAX сервер може повернути ці рядки XML, JSON або HTML.
Дані відповіді аналізуються за допомогою JavaScript.
Зрештою, проаналізовані дані оновлюються в DOM веб-сторінки.

Отже, як ви можете бачити, веб-сторінка оновлюється в реальному часі з сервера без перезавантаження браузера.
У наступному розділі ми розповімо, як реалізувати AJAX за допомогою ванільного JavaScript.

Як AJAX працює з використанням Vanilla JavaScript

У цьому розділі ми побачимо, як AJAX працює у звичайному JavaScript. Звичайно є доступні бібліотеки JavaScript, які полегшують виконання AJAX-дзвінків, але завжди цікаво знати, що відбувається всередині.

Давайте подивимося на наступний ванільний код JavaScript, який виконує виклик AJAX та асинхронно отримує відповідь від сервера.
            
                < script > _ _
                    var objXMLHttpRequest = новий  XMLHttpRequest ();
                    objXMLHttpRequest.onreadystatechange = функція()  {
                        if (objXMLHttpRequest.readyState === 4) {
                            if (objXMLHttpRequest.status === 200) {
                                alert(objXMLHttpRequest.responseText);
                            } else {
                                alert(' Error Code: ' + objXMLHttpRequest.status);
                                alert(' Error Message : ' + objXMLHttpRequest.statusText);
                            }
                        }
                    }
                    objXMLHttpRequest.open(' GET ', ' request_ajax_data.php ');
                    objXMLHttpRequest.send();
                    < / script >
Давайте пройдемося за наведеним вище кодом, щоб зрозуміти, що відбувається за лаштунками.< br >
Спочатку ми ініціалізуємо об'єкт XMLHttpRequest, який відповідає за виконання викликів AJAX.
Об'єкт XMLHttpRequestмає властивість readyState, і значення цієї властивості змінюється протягом життєвого циклу запиту. Воно може містити одне з чотирьох значень: OPENED, HEADERS_RECEIVED, LOADINGі DONE.
Ми можемо налаштувати функцію-обробник для зміни стану, використовуючи властивість onreadystatechange. І це те, що ми зробили в наведеному вище прикладі: ми використовували функцію, яка буде викликатись щоразу при зміні якості стану.
У цій функції ми перевірили, чи значення readyState 4, що означає, що запит завершено, і ми отримали відповідь від сервера. Потім ми перевірили, чи дорівнює код стану 200, що означає, що запит був успішним. Нарешті, ми отримуємо відповідь, яка зберігається як responseTextоб'єкт XMLHttpRequest.
Після налаштування обробника ми ініціюємо запит, викликаючи метод openоб'єкта XMLHttpRequest. Значення властивості readyStateбуде встановлено 1 після цього виклику.
Нарешті ми викликали метод sendоб'єкта XMLHttpRequest, який фактично надсилає запит на сервер. Значення властивості readyStateбуде встановлено 2 після цього виклику.
Коли сервер відповідає, він зрештою встановить значення readyStateрівним 4 і ви повинні побачити вікно попередження, що відображає відповідь від сервера.

Ось як AJAX працює з ванільним JavaScript. Звичайно, це був дуже простий приклад для демонстрації концепції AJAX, і в реальному додатку все може бути досить складно, тому що вам потрібно обробити багато різних сценаріїв успіху та невдач. Таким чином, було б непогано вибрати бібліотеку JavaScript, яка ховає специфічні складності браузера!
У наступному розділі ми побачимо, як використовувати бібліотеку jQuery для здійснення викликів AJAX.

Як працює AJAX з використанням бібліотеки jQuery

У попередньому розділі ми обговорювали, як можна здійснювати дзвінки AJAX, використовуючи ванільний JavaScript. У цьому розділі ми будемо використовувати бібліотеку jQuery, щоб продемонструвати це. Я припускаю, що ви знаєте основу бібліотеки jQuery.

Бібліотека jQuery надає кілька різних методів для виконання викликів AJAX, хоча тут ми розглянемо стандартний метод ajax, який найчастіше використовується.

Подивіться наступний приклад.

                                                                                
                                                                                    < script > _ _
                                                                                        $ . ajax (
                                                                                        ' request_ajax_data.php ' ,
                                                                                        {
                                                                                            success :  function ( data )  {
                                                                                            alert(' AJAX call was successful! ');
                                                                                        alert ( ' Data from the server '  +  data );
                                                                                      },
                                                                                        error :  function ()  {
                                                                                            alert(' There was some error performing the AJAX call! ');
                                                                                      }
                                                                                   }
                                                                                        );
                                                                                        < /script >
Як ви вже знаєте, знак $використовується для посилання на jQuery об'єкт.

Першим параметром методу ajaxє URL-адреса, яка буде викликатись у фоновому режимі для отримання вмісту з боку сервера. Другий параметр має формат JSON і дозволяє вказувати значення деяких різних параметрів, підтримуваних методом ajax.

У більшості випадків вам потрібно буде вказувати функції зворотного виклику (коллбеки) для успішного виконання та помилок. Функцію зворотного виклику для успіху буде викликано після успішного завершення виклику AJAX. Відповідь, повернена сервером, буде передана коллбеку для успіху. З іншого боку, коллбек для помилки буде викликаний, якщо щось піде не так, і виникла проблема під час виклику AJAX.

Отже, як ви можете бачити, операції AJAX легко виконувати за допомогою бібліотеки jQuery. Фактично, процес більш менш однаковий, незалежно від бібліотеки JavaScript, з якою ви вирішуєте здійснювати виклики AJAX.

Реальний приклад AJAX з PHP

У цьому розділі ми створимо приклад, який витягує контент JSON з PHP-файлу на стороні сервера, використовуючи AJAX.

Для демонстрації ми створимо приклад, який виконує вхід користувача за допомогою AJAX та jQuery. Спочатку давайте створимо файл index.php , як показано в наступному фрагменті, який відображає базову форму входу.

Файл index.php є досить стандартною HTML-формою, яка містить поля імені користувача та пароля. Він також містить фрагмент JavaScript jQuery, який слідує за схемою, яку ми бачили вище.

Ми використовували подію submitелемента форми, який спрацьовуватиме, коли користувач натискає кнопку відправки. У цьому обробнику подій ми ініціювали виклик AJAX, який надсилає дані форми у файл login.php , використовуючи метод POST асинхронно. Отримавши відповідь сервера, ми аналізуємо його, використовуючи метод parseоб'єкта JSON. І нарешті, ґрунтуючись на успіху чи невдачі, ми робимо відповідні дії.
