Абстрактні класи та методи

Абстрактний клас представляє часткову реалізацію для класів-спадкоємців. Абстрактний клас визначається за допомогою модифікатора abstract , який ставиться перед ім’ям класу: 
            
                abstract class Messenger { }
Однією з ключових особливостей абстрактних класів є те, що ми не можемо створити об’єкти абстрактного класу безпосередньо за допомогою виклику його конструктора: 
            
                abstract class Messenger { }
                $telegram = new Messenger(); // цей рядок не працюватиме
Абстрактні класи, як і звичайні класи, можуть визначати змінні та константи, методи та конструктори. Але іншою особливістю абстрактних класів і те, що можуть містити абстрактні методи. Це методи, що не мають реалізації. Реалізацію їм надають класи-спадкоємці. Наприклад: 
            
                abstract class Messenger
                {
                    abstract function send( $message );   // абстрактний метод
                }
Під час визначення абстрактного методу перед словом function ставиться модифікатор abstract . А після списку параметрів методу – точка з комою. Абстрактні методи можуть бути розміщені тільки в абстрактних класах. Звичайний неабстрактний клас неспроможна мати абстрактних методів.
Якщо неабстрактний клас успадковується від абстрактного класу, він повинен реалізувати всі його абстрактні методи.
Для успадкування класом абстрактного класу, як і у випадку при успадкування, застосовується ключове слово extends . Наприклад: 
            
                abstract class Messenger
                {
                    protected $name;
                     
                    function __construct( $name )
                    {
                        $this->name = $name;
                    }
                     
                    abstract function send( $message );
                     
                    function close()
                    {
                        echo "Вихід із месенджера...";
                    }
                }
                 
                class EmailMessenger extends Messenger 
                {
                    function send( $message ) 
                    {
                        echo "$this->name надсилає повідомлення: $message
";
                    }
                }
                 
                $outlook = new EmailMessenger( "Outlook" );
                $outlook->send( "Hello PHP 8" );
                $outlook->close();
У разі клас EmailMessenger успадковується від абстрактного класу Messenger. Абстрактний клас визначає абстрактний метод send(), тому клас-спадкоємець EmailMessenger зазвичай повинен надати реалізацію для цього методу. Так, у цьому випадку ми отримаємо наступний висновок: 
            
                Outlook надсилає повідомлення: Hello PHP 8
                Вихід із месенджера...
Можна зауважити, що абстрактні класи схожі на інтерфейси – тому що вони обидва можуть визначати методи без реалізації, що реалізуються в інших класах. Однак абстрактні класи, як і звичайні класи, можуть мати змінні, неабстрактні методи, конструктори, а інтерфейси немає. Крім того, в PHP один клас може успадковуватись тільки від одного класу, тоді як один клас може застосовувати відразу кілька інтерфейсів.
